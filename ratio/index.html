Ratio

Ratio⁡(x,y)=exp⁡(Shift⁡(log⁡x,log⁡y))\operatorname{Ratio}(\mathbf{x}, \mathbf{y}) = \exp(\operatorname{Shift}(\log \mathbf{x}, \log \mathbf{y}))Ratio(x,y)=exp(Shift(logx,logy))

Robust measure of scale ratio between two samples — the multiplicative dual of Shift⁡\operatorname{Shift}Shift.

  • Asymptotic — geometric median of pairwise ratios xiyj\frac{x_i}{y_j}yj​xi​​ (via log-space aggregation)
  • Domain — xi>0x_i > 0xi​>0, yj>0y_j > 0yj​>0
  • Assumptions — positivity(x), positivity(y)
  • Unit — dimensionless
  • Complexity — O((m+n)log⁡L)O((m + n) \log L)O((m+n)logL)

Properties

  • Self-ratio Ratio⁡(x,x)=1\operatorname{Ratio}(\mathbf{x}, \mathbf{x}) = 1Ratio(x,x)=1
  • Scale equivariance Ratio⁡(kx⋅x,ky⋅y)=(kxky)⋅Ratio⁡(x,y)\operatorname{Ratio}(k_x \cdot \mathbf{x}, k_y \cdot \mathbf{y}) = (\frac{k_x}{k_y}) \cdot \operatorname{Ratio}(\mathbf{x}, \mathbf{y})Ratio(kx​⋅x,ky​⋅y)=(ky​kx​​)⋅Ratio(x,y)
  • Multiplicative antisymmetry Ratio⁡(x,y)=1/Ratio⁡(y,x)\operatorname{Ratio}(\mathbf{x}, \mathbf{y}) = 1 / \operatorname{Ratio}(\mathbf{y}, \mathbf{x})Ratio(x,y)=1/Ratio(y,x)

Example

  • Ratio([1, 2, 4, 8, 16], [2, 4, 8, 16, 32]) = 0.5
  • Ratio(x, x) = 1 Ratio(2x, 5y) = 0.4 · Ratio(x, y)

Relationship to Shift

Ratio⁡\operatorname{Ratio}Ratio is the multiplicative analog of Shift⁡\operatorname{Shift}Shift. While Shift⁡\operatorname{Shift}Shift computes the median of pairwise differences xi−yjx_i - y_jxi​−yj​, Ratio⁡\operatorname{Ratio}Ratio computes the median of pairwise ratios xiyj\frac{x_i}{y_j}yj​xi​​ via log-transformation. This relationship is expressed formally as:

Ratio⁡(x,y)=exp⁡(Shift⁡(log⁡x,log⁡y))\operatorname{Ratio}(\mathbf{x}, \mathbf{y}) = \exp(\operatorname{Shift}(\log \mathbf{x}, \log \mathbf{y}))Ratio(x,y)=exp(Shift(logx,logy))

The log-transformation converts multiplicative relationships to additive ones, allowing the fast Shift⁡\operatorname{Shift}Shift algorithm to compute the result efficiently. The exp-transformation converts back to the ratio scale.

Ratio⁡\operatorname{Ratio}Ratio is appropriate for multiplicative relationships rather than additive differences. If one system is twice as fast or prices are 30% lower, the underlying thinking is in ratios. A result of 0.5 means the first group is typically half the size of the second; 2.0 means twice as large. This estimator is appropriate for quantities like prices, response times, and concentrations where relative comparisons make more sense than absolute ones. Both samples must contain strictly positive values.

Algorithm

The Ratio⁡\operatorname{Ratio}Ratio estimator measures the typical multiplicative relationship between elements of two samples. Given samples x=(x1,x2,…,xn)\mathbf{x} = (x_1, x_2, \ldots, x_n)x=(x1​,x2​,…,xn​) and y=(y1,y2,…,ym)\mathbf{y} = (y_1, y_2, \ldots, y_m)y=(y1​,y2​,…,ym​) with all positive values, this estimator is defined as:

Ratio⁡(x,y)=exp⁡(median⁡i,j(log⁡xi−log⁡yj))=exp⁡(Shift⁡(log⁡x,log⁡y))\operatorname{Ratio}(\mathbf{x}, \mathbf{y}) = \exp(\operatorname{median}_{i,j}(\log x_i - \log y_j)) = \exp(\operatorname{Shift}(\log \mathbf{x}, \log \mathbf{y}))Ratio(x,y)=exp(mediani,j​(logxi​−logyj​))=exp(Shift(logx,logy))

A naive approach would compute all n×mn \times mn×m ratios, sort them, and extract the median. With n=m=10000n = m = 10000n=m=10000, this creates 100 million values, requiring quadratic memory and O(nmlog⁡(nm))O(n m \log(n m))O(nmlog(nm)) time.

The presented algorithm avoids this cost by exploiting the multiplicative-additive duality. Taking the logarithm of a ratio yields a difference:

log⁡(xiyj)=log⁡(xi)−log⁡(yj)\log(\frac{x_i}{y_j}) = \log(x_i) - \log(y_j)log(yj​xi​​)=log(xi​)−log(yj​)

This transforms the problem of finding the median of pairwise ratios into finding the median of pairwise differences in log-space.

The algorithm operates in three steps:

  • Log-transform — Apply log⁡\loglog to each element of both samples. If x\mathbf{x}x was sorted, log⁡x\log \mathbf{x}logx remains sorted (log is monotonically increasing for positive values).

  • Delegate to Shift — Use the Shift algorithm to compute the desired quantile of pairwise differences in log-space. This leverages the O((m+n)log⁡L)O((m + n) \log L)O((m+n)logL) complexity of the Shift algorithm.

  • Exp-transform — Apply exp⁡\expexp to convert the result back to ratio-space. If the log-space median is ddd, then exp⁡(d)=exp⁡(log⁡(xi)−log⁡(yj))=xiyj\exp(d) = \exp(\log(x_i) - \log(y_j)) = \frac{x_i}{y_j}exp(d)=exp(log(xi​)−log(yj​))=yj​xi​​ is the original ratio.

The total complexity is O((m+n)log⁡L)O((m + n) \log L)O((m+n)logL) per quantile, where LLL represents the convergence precision in the log-space binary search. This is dramatically more efficient than the naive O(nmlog⁡(nm))O(n m \log(n m))O(nmlog(nm)) approach.

Memory usage is O(m+n)O(m + n)O(m+n) for storing the log-transformed samples, compared to O(nm)O(n m)O(nm) for materializing all pairwise ratios.

namespace Pragmastat.Algorithms;

using System;
using System.Collections.Generic;
using System.Linq;
using Pragmastat.Exceptions;
using Pragmastat.Internal;

/// <summary>
/// Computes quantiles of pairwise ratios via log-transformation and FastShift delegation.
/// Ratio(x, y) = exp(Shift(log(x), log(y)))
/// </summary>
public static class FastRatio
{
  /// <summary>
  /// Computes quantiles of all pairwise ratios { x_i / y_j }.
  /// Time: O((m + n) * log(precision)) per quantile. Space: O(m + n).
  /// </summary>
  /// <remarks>
  /// Log-transformation preserves sort order for positive values.
  /// </remarks>
  /// <param name="x">First sample (must be strictly positive).</param>
  /// <param name="y">Second sample (must be strictly positive).</param>
  /// <param name="p">Probabilities in [0, 1].</param>
  /// <param name="assumeSorted">If true, assumes x and y are already sorted in ascending order.</param>
  /// <returns>Quantile values for each probability in p.</returns>
  public static double[] Estimate(IReadOnlyList<double> x, IReadOnlyList<double> y, double[] p, bool assumeSorted = false)
  {
    // Log-transform both samples (includes positivity check)
    var logX = MathExtensions.Log(x, Subject.X);
    var logY = MathExtensions.Log(y, Subject.Y);

    // Delegate to FastShift in log-space
    var logResult = FastShift.Estimate(logX, logY, p, assumeSorted);

    // Exp-transform back to ratio-space
    return logResult.Select(v => Math.Exp(v)).ToArray();
  }
}
